60//----------------------------------------------------------------------------
61// Conventions:
62//----------------------------------------------------------------------------
63// Naming convention:
64// cr_ prefix (for Cygnus Reach), then service name, then verb.
65// crcb_ prefix for the "weak" callback functions required.
66//
67// All API functions return zero on success or an identifiable error code.
68//
69// There are a small number of API functions that are to be called by
70// the hosting application. Most of the functions in the API are to be
71// provided by the application. The reach stack provides "weak" declared
72// versions so the application provide only what is required.
73// The stack can be built to report errors if required weak functions are
74// not overridden.
75//
76// The configuration structures that must be passed in to the stack to
77// describe the system are designed to be constant and stored in flash
78// if the application desires. This allows for a minimum use of RAM.
79//
80// Lists of items such as parameters, command, files, etc are handled using
81// an iterator pattern. A function gets the next item and a reset function
82// is available.
83// In all cases, resetting to zero will select the lowest available object.
84//
85
86//----------------------------------------------------------------------------
87// API
88//----------------------------------------------------------------------------
89
90// Error codes are defined in the proto file so that the client can see them.

129// The cr_process function calls crcb_get_coded_prompt()
130// to get any available prompt in coded format.
131// The implementation copies the data into the provided buffer and sets the size.
132// cr_store_coded_prompt() can be used to push data into the stack.
133// Then crcb_get_coded_prompt() can remain in its weak state, as the data
134// is already where it belongs.
135// The len variable must specify the number of valid bytes in the supplied prompt.
136// crcb_get_coded_prompt() must not block as that would disable any notifications.
137// Return cr_ErrorCodes_NO_ERROR if prompt data is supplied.
138// Return cr_ErrorCodes_NO_DATA if no data is available.
139// A message larger than CR_CODED_BUFFER_SIZE is fatal.
140int crcb_get_coded_prompt(uint8_t *prompt, size_t *len);
141
142// allows the app to store the coded prompt in the memory held by the stack.
143int cr_store_coded_prompt(uint8_t *data, size_t len);
